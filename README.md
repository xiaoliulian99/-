## 📖 项目概述

**打造基于飞书 AI 的设备智能运维中台，让 AI 管理的不只是工单，更是设备知识、维修决策和组织协同——实现从"事后维修"向"预测运维"的转变。**

---

## 🎯 我要解决的问题

野人先生 1300 家门店，核心设备是意大利进口 Gelato 凝冻机。店员说不清故障 → 维修师傅带错配件 → 供应商信息滞后 → 三方各说各话。

这不是"缺一个报修系统"，而是缺**维修决策能力**——师傅出发之前，没人告诉他故障根因、带什么配件、能不能一次修好。

## 🔑 四个核心创新点

| # | 创新点 | 一句话 |
|:--:|------|------|
| ① | **RAG 诊断引擎** | 四重数据（历史工单 + 同型号故障 + 维修手册 + 案例标签）交叉验证，让 AI 告诉师傅"带什么配件、一次修好" |
| ② | **双模式预测** | 不假设设备都已 IoT 联网——联网设备走 API 实时监测，存量设备靠历史工单做风险评分，当前就能落地 |
| ③ | **四 Agent 协同** | 智报·智修·智供·智管在飞书多维表格上原生运行，打破门店/维修/供应商/管理层信息孤岛 |
| ④ | **持续学习闭环** | 故障 → 诊断 → 维修 → AI 复盘 → 知识库更新 → 下一次更准——数据越积累，AI 越聪明 |

## 🛠️ 技术栈

全部基于 **飞书 AI 原生能力**，零额外软件采购：

| 能力 | 飞书产品 | 用途 |
|------|----------|------|
| AI 字段 | 多维表格 | 口语→标准工单、全国故障聚合分析 |
| AI Agent | Aily 智能伙伴 | 故障诊断、知识库 RAG 问答 |
| RAG 检索 | 知识库 + Aily | 维修手册/案例智能检索 |
| OCR | 飞书图片识别 | 拍照提取设备铭牌型号 |
| AI 总结 | AI 智能总结 | 维修复盘报告、日报自动生成 |
| 消息卡片 | 飞书消息 | 供应商任务卡、门店进度推送 |
| 自动化 | 飞书自动化 | Agent 间信息传递引擎 |
| 仪表盘 | 多维表格仪表盘 | 管理驾驶舱 |
| API 对接 | 开放平台 HTTP 连接器 | 设备 IoT 数据接入（进阶） |
